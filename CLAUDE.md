# Toxic Comment Detection in Russian Social Media Text

## Project Overview

Binary classification of Russian-language social media comments as toxic or non-toxic. Compares **6 model families**: three classical TF-IDF baselines (Multinomial Naive Bayes, Logistic Regression, Linear SVM) and three transformers (ruBERT, mBERT, XLM-RoBERTa). The paper positions itself as the **first head-to-head comparison of ruBERT, mBERT, and XLM-RoBERTa** on the standard Kaggle Russian Toxic Comments benchmark, grounded by a full classical-baseline trio. Research paper formatted with Springer Nature LaTeX template.

## Dataset

- **Source:** Kaggle Russian Language Toxic Comments (2ch.hk + Pikabu.ru)
- **File:** `data/labeled.csv` — CSV with columns `comment` (text) and `toxic` (0.0/1.0)
- **Size:** ~14,412 valid comments after cleaning (raw file has ~27K lines due to multiline comments)
- **Class split:** ~66.5% non-toxic, ~33.5% toxic
- **Language:** Russian, informal, contains profanity and slurs
- **Download:** https://www.kaggle.com/datasets/blackmoon/russian-language-toxic-comments — place `labeled.csv` in `data/`

## How to Run (Step by Step)

### 1. Setup

```bash
pip install -r requirements.txt
```

Required: Python 3.10+, pandas, numpy, matplotlib, seaborn, wordcloud, scikit-learn, transformers, torch, datasets, accelerate, pyarrow.

### 2. Preprocessing — Generate train/val/test splits

```bash
python3 src/preprocessing.py
```

This reads `data/labeled.csv`, cleans text (lowercase, remove URLs/mentions, normalize whitespace), and creates stratified 70/15/15 splits:
- `data/train.csv` (10,088 rows)
- `data/val.csv` (2,162 rows)
- `data/test.csv` (2,162 rows)

All splits preserve the ~33.5% toxic class ratio. Random seed: 42.

**You must run this before any notebook.**

### 3. EDA — `notebooks/01_eda.ipynb`

Exploratory data analysis. Run all cells. Produces:
- Class distribution chart
- Comment length histograms (chars + tokens) by class
- Top-20 word frequency bar charts per class
- Word clouds per class
- Sample comments
- Summary statistics table

**Note:** If word clouds show boxes instead of Cyrillic, uncomment the `font_path` line in the word cloud cell and set it to a Cyrillic-capable font on your system.

### 4. Baselines — `notebooks/02_baselines.ipynb`

Trains and evaluates three TF-IDF classical baselines. Run all cells. Produces:
- Multinomial Naive Bayes: TF-IDF (50K features, bigrams, sublinear_tf) + MultinomialNB (alpha=1.0) — replicates Smetanin (2020) classical baseline
- Logistic Regression: TF-IDF + LogReg (C=1.0, lbfgs)
- Linear SVM: TF-IDF + LinearSVC with CalibratedClassifierCV (for probability estimates)
- Confusion matrices, ROC curves, results comparison table
- Saves `results/baseline_metrics.json`

Expected F1 (toxic class): MNB ~0.70-0.78, LogReg ~0.75-0.80, SVM ~0.78-0.83.

### 5. Transformers — `notebooks/03_transformers.ipynb`

Fine-tunes ruBERT, mBERT, and XLM-RoBERTa. Run all cells. Produces:
- ruBERT (`DeepPavlov/rubert-base-cased`) — 3 epochs, batch 8, grad accum 2, lr 2e-5, max_length 128
- mBERT (`bert-base-multilingual-cased`) — same hyperparameters
- XLM-RoBERTa (`xlm-roberta-base`) — same hyperparameters
- Training loss curves, validation metrics per epoch
- Confusion matrices, ROC curves, results comparison (head-to-head)
- Saves `results/transformer_metrics.json`, `results/test_predictions.csv`, model checkpoints to `models/`

Expected F1 (toxic class): mBERT ~0.88-0.91, ruBERT ~0.90-0.92, XLM-RoBERTa ~0.90-0.93.

**Hardware notes:**
- GPU recommended. Works on Apple M1/M2 (MPS), NVIDIA GPU (CUDA), or CPU (slow).
- M1 with 8GB RAM: use batch_size=8, max_length=128 (already configured).
- If you get MPS out-of-memory errors, reduce BATCH_SIZE to 4.

### 6. Error Analysis — `notebooks/04_error_analysis.ipynb` (planned)

Will analyze misclassified examples across all models.

## Key Technical Details

- **Random seed:** 42 everywhere (numpy, sklearn, torch, transformers)
- **Text cleaning:** Minimal — lowercase, remove URLs, remove @mentions, normalize whitespace. Does NOT remove Cyrillic, punctuation, or emoji (preserves signal for models).
- **TF-IDF config:** max_features=50000, ngram_range=(1,2), sublinear_tf=True
- **Transformer config:** max_length=128, epochs=3, batch_size=8, gradient_accumulation=2 (effective batch=16), lr=2e-5, weight_decay=0.01, best model by val F1 macro
- **Keras/TF conflict fix:** `src/transformers_train.py` sets `USE_TF=0` to prevent transformers from loading TensorFlow/Keras. If you still get Keras 3 errors, run `pip uninstall tensorflow keras -y`.

## Module Reference

### `src/preprocessing.py`
- `clean_text(text)` — clean a single string
- `load_and_clean(filepath)` — load CSV, clean, return DataFrame with `clean_comment` column
- `split_data(df, output_dir)` — stratified split, saves CSVs, returns (train, val, test) DataFrames

### `src/baselines.py`
- `build_mnb_pipeline(max_features, ngram_range)` — returns sklearn Pipeline (TF-IDF + MultinomialNB)
- `build_logreg_pipeline(max_features, ngram_range)` — returns sklearn Pipeline
- `build_svm_pipeline(max_features, ngram_range)` — returns sklearn Pipeline with calibration

### `src/evaluation.py`
- `compute_metrics(y_true, y_pred, y_prob)` — returns dict with accuracy, precision, recall, F1, ROC-AUC
- `print_report(y_true, y_pred)` — prints sklearn classification report
- `plot_confusion_matrix(y_true, y_pred, title, ax)` — heatmap
- `plot_roc_curve(y_true, y_prob, label, ax)` — ROC curve
- `save_metrics(metrics_dict, filepath)` — save to JSON

### `src/transformers_train.py`
- `ToxicDataset(texts, labels, tokenizer, max_length)` — PyTorch Dataset
- `load_splits(data_dir)` — load train/val/test CSVs, return dict
- `compute_metrics_hf(eval_pred)` — HuggingFace Trainer compatible
- `build_model_and_tokenizer(model_name)` — load pretrained model + tokenizer
- `get_training_args(output_dir, ...)` — standard TrainingArguments
- `get_predictions(trainer, dataset)` — returns (preds, probs, labels)
